# 数探｜智能数据分析 Agent

## 产品需求说明书（投递精简版）

| 项目 | 内容 |
| --- | --- |
| 产品定位 | 面向业务人员的对话式 AI 数据分析工作台 |
| 核心用户 | 使用 CSV/Excel 进行经营分析与临时取数、但缺少 SQL 与分析方法的运营及业务人员 |
| 核心价值 | 让自然语言问题获得来自真实数据、可查看依据的分析结果 |

## 1. 问题与机会

业务人员能提出“哪个渠道转化更高”“本月销售为何变化”等问题，但完成答案仍要经过文件整理、字段理解、指标确认、查询、图表和汇报。Excel 操作成本高，SQL 依赖专业技能，BI 更适合固定看板，通用模型则难以让用户判断结果是否真正来自当前数据。

| 研究信号 | 产品机会 |
| --- | --- |
| Microsoft 与 LinkedIn 的 [2024 Work Trend Index](https://www.microsoft.com/en-us/worklab/work-trend-index/2024/) 显示：31,000 名知识工作者中，75% 已在工作中使用 AI；AI 使用者中，78% 自带工具。 | 用户已习惯以自然语言发起工作任务，产品需要把这种交互连接到业务数据。 |
| McKinsey 的 [The state of AI in early 2024](https://www.mckinsey.com/capabilities/quantumblack/our-insights/the-state-of-ai) 显示：65% 的受访组织已在至少一个业务职能中经常使用生成式 AI。 | 差异化不在于“能否聊天”，而在于能否完成可复核的数据执行。 |

**核心矛盾：**业务人员会提问题，但缺少 SQL、数据处理与分析方法，也难以判断通用模型的结论是否基于真实数据。

## 2. 产品定义

数探将数据接入、自然语言提问、工具执行、结果核验和后续追问放入一个工作台。大模型负责理解、规划和解释；确定性工具负责查询、计算、清洗和图表生成。

## 3. 目标用户与核心场景

| 场景 | 用户任务 | 交付结果 |
| --- | --- | --- |
| 经营复盘 | 上传销售、活动或渠道文件，定位趋势与差异。 | 汇总表、趋势图、结论。 |
| 临时取数 | 用自然语言完成分组、筛选、Top N、比较。 | 可下载的数据表与计算摘要。 |
| 数据质量检查 | 检查缺失、重复、异常值和数据覆盖范围。 | 质量摘要与问题字段。 |
| 口径复用 | 按“有效客户”“复购率”等既定规则继续分析。 | 知识库引用、连续追问与历史分析。 |

## 4. 产品策略

| 设计原则 | 设计落点 |
| --- | --- |
| 自动执行优先 | 数据和口径充分时直接执行；仅对时间范围、指标口径、关联键等高影响歧义发起确认。 |
| 真实执行优先 | 数值、排名、图表来自 SQL、统计、清洗和图表工具的真实结果。 |
| 过程可追溯 | 结果展示数据范围、工具步骤、SQL/计算摘要、表格或图表来源。 |
| 不展示内部推理 | 向用户展示计划、状态和执行摘要，不展示模型内部思维链。 |
| 失败显式反馈 | 解析、工具、超时和权限失败提供状态、原因与下一步入口。 |

## 5. MVP 与版本边界

| 优先级 | 范围 | 成功判断 |
| --- | --- | --- |
| P0 | CSV/Excel 上传、Sheet/字段/样例预览、自然语言提问、查询/清洗/统计/图表、结果依据、多轮追问、取消与重试。 | 用户可完成“文件 → 分析 → 核验 → 追问”的闭环。 |
| P1 | SQL/Google Sheets/HTTP API、工作区、导出、后台任务、登录、历史、个人知识库与偏好。 | 分析结果和业务口径可保存、恢复和复用。 |
| P2 | MCP、Teams、团队知识库、协作权限、定时任务与主动异常发现。 | 面向组织级协作与自动化。 |

## 6. 核心流程与功能验收

```mermaid
flowchart LR
    A[添加数据] --> B[预览字段与样例]
    B --> C{关键歧义?}
    C -- 否 --> D[提出问题并执行]
    C -- 是 --> E[确认口径或范围]
    E --> D --> F[表格、图表与结论] --> G[追问、导出或保存]
```

| 功能 | 关键规则 | 验收标准 |
| --- | --- | --- |
| 文件与数据预览 | 支持 `.csv`、`.xlsx`、`.xls`；单次最多 5 个文件；展示 Sheet、字段、类型与样例。 | 文件解析成功后才能进入分析；空文件、损坏文件和无可用 Sheet 有明确提示。 |
| 对话分析 | 明确问题直接执行；关键歧义才询问。 | 追问继承数据范围与已确认口径；用户可停止生成。 |
| 工具执行 | SQL 只读校验，参数由 Schema 校验。 | 数据结论可回溯至工具结果；工具失败不伪造成功。 |
| 结果交付 | 输出结论、表格、图表、数据范围和执行摘要。 | 图表/导出失败时保留已成功结果并标记任务状态。 |
| 知识与历史 | 个人知识库保存指标定义、规则和背景知识；会话支持保存与恢复。 | 用户数据、历史和知识库相互隔离。 |

## 7. 责任边界

| 能力 | 大模型 | 工具/系统 | 用户 |
| --- | --- | --- | --- |
| 理解问题 | 理解意图、拆解任务。 | 提供数据结构与可用工具。 | 提供业务目标。 |
| 确定口径 | 识别歧义、提出选项。 | 校验字段和数据支持度。 | 确认关键口径。 |
| 查询与计算 | 生成调用参数、组织步骤。 | 实际执行并返回结果。 | 查看结果。 |
| 业务结论 | 基于工具结果解释并标记不确定性。 | 提供真实证据与安全校验。 | 对高风险决策进行业务复核。 |

## 9. 指标与评测

| 指标类别 | 核心指标 |
| --- | --- |
| 产品价值 | 首次分析成功率、首个有效结果耗时、有效任务完成率、结果采纳率、多轮追问率、历史与口径复用率。 |
| Agent 质量 | 工具路由正确率、SQL 执行/结果正确率、图表适配率、任务完成率、数据幻觉率、安全拒绝正确率。 |
| 效率 | 工具调用次数、P50/P95 耗时、Token、单任务成本、达到最大轮次比例。 |

`benchmark_v1` 包含 55 条中文业务问题，并按固定种子拆分为 35 条开发集和 20 条独立测试集。独立测试中，Prompt B 相比 Prompt A：SQL 结果正确率从 43.75% 提升至 50.00%，端到端任务完成率从 50.00% 提升至 55.00%，图表合适率均为 100.00%。

## 10. 当前证据与限制

离线评测使用匿名演示数据，覆盖查询、趋势、图表、质量检查、清洗、K-Means、安全拒答和因果边界。当前三个工作簿为单表，JOIN、多数据源关联和个人知识库不计入量化得分；每个独立测试案例运行一次，因此结果用于方向性比较，不用于统计显著性判断。
